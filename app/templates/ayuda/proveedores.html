Gestión de proveedores

El catálogo de proveedores es compartido por todos los servicios de FGULEM. Un proveedor solo necesita crearse una vez, y todos los servicios pueden usarlo en sus PEGs. Los datos bancarios del proveedor se usan directamente para generar el fichero Cuaderno 34.

1. El catálogo de proveedores

Todos los usuarios pueden buscar en el catálogo. Puedes crear proveedores nuevos desde el formulario del PEG (modal rápido) o desde el menú «Proveedores → Nuevo proveedor» (formulario completo con todos los campos).

Antes de crear un proveedor nuevo, busca por CIF/NIF para asegurarte de que no existe ya. El sistema impide duplicados de CIF, pero podrías encontrarlo con una variación del nombre.
2. Buscar un proveedor desde el formulario de PEG

El campo Proveedor del formulario de PEG tiene un buscador en tiempo real. Empieza a escribir el nombre o el CIF/NIF del proveedor y aparecerán sugerencias automáticamente. Selecciona el que corresponda.

Consejo: busca por los primeros dígitos del CIF (ej. «B47») para encontrar proveedores aunque no recuerdes exactamente el nombre.
3. Crear un proveedor rápido (desde el PEG)

Si el proveedor no existe, haz clic en «+ Nuevo proveedor» junto al buscador. Se abrirá un modal con los campos mínimos necesarios para continuar:

  • CIF/NIF (obligatorio, único)
  • Razón social (obligatorio)
  • IBAN (obligatorio si el pago es por transferencia)
  • Email (para notificaciones)

El proveedor quedará guardado en el catálogo y podrás completar el resto de sus datos más adelante desde el menú de Proveedores.

4. Crear un proveedor completo

Ve a Proveedores → Nuevo proveedor para el formulario completo. Es la opción recomendada cuando vas a crear un proveedor con el que trabajarás de forma recurrente.

5. Campos del proveedor — referencia completa
CampoDescripción y notas importantes
CIF / NIF * Identificador fiscal del proveedor. Para empresas: CIF (letra + 8 dígitos). Para autónomos: NIF (8 dígitos + letra) o NIE. Debe ser único en el sistema.
Razón social * El nombre legal completo tal como aparece en la factura y en el registro mercantil. Para autónomos, nombre y apellidos.
Nombre comercial El nombre con el que se conoce habitualmente al proveedor si es diferente de la razón social. Ej: razón social «García López, Juan» → nombre comercial «Estudio GráficoJG».
Tipo de proveedor * Define cómo se realiza el pago:
Transferencia: pago bancario SEPA (el más habitual, requiere IBAN)
Domiciliación: el cargo lo inicia el proveedor desde su banco
Caja: pago en efectivo
Mixto: puede usar más de un método según la factura
IBAN * Número de cuenta bancaria internacional (ES + 22 dígitos). Obligatorio para proveedores de tipo Transferencia. Este número se incluye directamente en el fichero Cuaderno 34 que se envía al banco. Un IBAN incorrecto provoca la devolución del pago.
BIC / SWIFT Código identificador del banco del proveedor (8 u 11 caracteres). Recomendado para pagos internacionales. Para cuentas españolas generalmente no es necesario.
Email Dirección de correo del proveedor o su persona de contacto. SGPEG puede usarla para enviar confirmaciones de pago si está configurado.
Teléfono Dato de contacto para incidencias de facturación.
Dirección Dirección fiscal del proveedor. Útil para verificar la identidad en caso de dudas.
Sujeto a IRPF Activa si las facturas de este proveedor llevan retención de IRPF. Se activa automáticamente para autónomos que prestan servicios profesionales. Al activarlo, debes indicar también el tipo de retención.
Tipo IRPF Porcentaje de retención: 7% (inicio de actividad), 15% (retención general de profesionales), 19% (arrendamientos de bienes). Consulta la factura del proveedor para ver qué tipo aplica.
Cuenta cliente A3Con Código de cuenta en el plan contable de A3Con, en el grupo 410 (acreedores por prestación de servicios) o 400 (proveedores). Formato: 410XXXXX. Solo visible y editable por GE y Admin. Se usa para generar el fichero SUENLACE.DAT de exportación a A3Con.
6. La importancia del IBAN correcto

El IBAN del proveedor se incluye directamente en el fichero Cuaderno 34 que se sube al banco. Si el IBAN es incorrecto:

  • El banco devolverá la transferencia (devolución SEPA)
  • El PEG quedará como pagado en SGPEG pero el proveedor no habrá cobrado
  • Será necesario repetir el proceso en una nueva remesa
  • El proveedor puede reclamar intereses de demora
Verifica siempre el IBAN directamente con el proveedor antes de introducirlo en el sistema. Pide un correo de confirmación o un documento oficial con el IBAN.
7. Cuenta A3Con — qué es y cómo funciona

La cuenta cliente A3Con es el código que identifica al proveedor en el software de contabilidad de FGULEM. Es un código del grupo 410 (acreedores prestación de servicios) del Plan General Contable. Ejemplos:

  • 4100001 — Proveedor nº 1 en A3Con
  • 4100142 — Proveedor nº 142 en A3Con

Cuando el GE exporta una remesa a A3Con (fichero SUENLACE.DAT), cada asiento contable incluye este código para que A3Con lo registre correctamente en la cuenta del proveedor.

Si el proveedor no tiene cuenta A3Con asignada, el Gestor Económico debe asignarla antes de cerrar la remesa o el fichero de exportación tendrá errores. El sistema muestra una alerta en este caso.
